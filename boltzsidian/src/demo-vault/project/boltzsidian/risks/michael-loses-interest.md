# Michael loses interest

The one item on this list that isn't code-fixable. A solo project
stops when the solo person stops.

Mitigations:

- Phase 3 is designed to be a usable standalone app. If the project
  halts there, the result is still a nicely-built graph-view notebook.
  No single phase is a cliff.
- Exit gates force confrontation with "do I still want this?" every
  week or two.
- Aesthetic parity with the sim at every phase — if the thing looks
  like a generic web app for a day, it starts feeling like a chore.

Early warning sign: more than two weeks between commits without a
stated reason.

Related: [[Michael]], [[Scope creep]].
